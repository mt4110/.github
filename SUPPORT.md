# Support

Read the repository `README.md` and local docs first, then use the path below that best matches the problem.

## Usage Question or Integration Help

If GitHub Discussions is enabled for the repository, use the closest Q&A or help category.

If Discussions is not enabled, open an issue with enough context for someone else to reproduce the question.

## Bug or Regression

Use the bug report form or the issue tracker.

Please include:

- package, app, or repository version when relevant
- environment details
- a minimal reproduction
- what you expected to happen

## Feature Request or Docs Gap

Use the feature request form when available.

Small, focused pull requests are also welcome. See [CONTRIBUTING.md](./CONTRIBUTING.md).

## Early Idea or Direction Discussion

Use GitHub Discussions when the shape of the request is still forming.

If Discussions is not enabled, open an issue that frames the problem, the use case, and any tradeoffs you already see.

## Security Concern

Do not open a public issue.

Follow [SECURITY.md](./SECURITY.md).

## Support Scope

Unless a repository says otherwise, there is no private support inbox or guaranteed response SLA.

Maintainer attention is focused on the repository itself and on improvements that can be shared back with other users.
