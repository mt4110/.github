# mt4110

Small, focused tools for the web.

The work here aims for software that feels calm in daily use:

- clear public surfaces
- durable defaults
- explicit release flows
- documentation that stays close to the work

## Repositories

- package repos for small reusable building blocks
- shared community defaults in `.github`
- starter repos for new package release lanes

## Working Style

- keep responsibility boundaries visible
- prefer boring automation over hidden framework logic
- let each repository own the settings and release details that are truly local
